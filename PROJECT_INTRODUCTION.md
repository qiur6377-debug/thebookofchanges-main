# 易经算命网站 (The Book of Changes) 项目介绍

## 1. 项目概述
这是一个基于现代 Web 技术与 AI 大语言模型相结合的“国学易经算命”全栈应用。项目通过数字化的方式模拟了传统的起卦过程（支持机器“快速起卦”与传统的“蓍草起卦法”），在推演出本卦、动爻、之卦（变卦）后，调用阿里云“通义千问”（DashScope API），通过流式输出（SSE）为用户提供分为古文剖析和现代白话文指导的双版本深度解析。

## 2. 核心架构与技术栈
- **后端服务**: `Node.js` + `Express` 提供静态文件托管及核心业务 API。
- **前端交互**: 原生 `JavaScript` (Vanilla JS) + 原生 `CSS`，没有任何繁重的前端框架负担。
- **数据持久化**: `SQLite` (sqlite3) 本地轻量级数据库，用于持久化用户的占卜历史。
- **AI 引擎核心**: 接入阿里云的大模型 (qwen-plus 模型)，根据内置专属于“周易国学大师”的角色 Prompt 提供解读。
- **本地知识树**: 项目内置由 `markdown-it` 解析的 64 卦知识体系文件。
- **文档体系**: 内置 `vitepress`，可通过相关命令启动本地静态文档预览。

## 3. 核心目录与文件职责
- **`server.js`**: 后端核心入口。
  - 初始化时，将根目录下 `zhouyi_1.md` - `zhouyi_64.md` 的本地 64 卦图文知识库载入内存。
  - 路由 `POST /api/divine`: 处理起卦逻辑。既支持后端代码通过 Math.random() 模拟掷爻产生卦象，也支持直接接收前端传递过来的“蓍草过程”爻值从而生成卦象。
  - 路由 `POST /api/interpret`: 后续的 AI 解析接口。接收用户问题与生成的卦象信息，组装复杂的周易 Prompt 推送至 LLM。同时以 SSE 的方式一边请求模型一边吐给前端达到“打字机”效果。最后会将问卦记录写入 SQLite 数据库。
- **`public/` (前端源码目录)**:
  - `index.html`: 用户所见的主界面页面。
  - `app.js` & `style.css`: 处理首屏交互、流式打字回显逻辑以及快速占卜功能的渲染。
  - `yarrow.js` & `yarrow.css`: 独立实现了复杂的“蓍草起卦法”沉浸式交互及计算，将纯手工的“大衍之数”起卦法进行了数字化还原。
- **`database.sqlite`**: 轻量数据库。其中具有 `divination_history` 表结构，会保存所有的提问问题、生成的卦号、动爻位置、生成的变卦以及 AI 直接给出的解语文本。
- **`.env`**: 存放私密配置的地方，通常用来存储 `DASHSCOPE_API_KEY`。

## 4. 业务使用流程梳理
1. **输入意境**: 用户在平台输入心中所虑之事（或心中默念）。
2. **选择起卦**:
   - 【普通快速起卦】：一键直达，直接调用后端随机概率算法完成“分二挂一揲四归奇”。
   - 【蓍草起卦】：进入高阶拟真模式，依靠用户在前端页面的手动分拣触发 6×3=18 变推演得到完整爻象。
3. **卦象计算与解析**: 系统锁定本卦（第几卦）、识别到是否有老阴老阳引发的“动爻”，进而得到其未来的走向表现“变卦”。
4. **AI 降临**: 前端将这些基础爻盘送入 `api/interpret` 大模型口，后端组装完毕后让通义千问以“国学大师”语气流式下吐文字：第一段为高雅周易辞意还原，第二段转入极简接地气的现代行动指南。
5. **归档落表**: 回复成型后，此测算的原始信息被静默存储进 SQLite。

## 5. 开发与部署命令
- `npm start`: 使用 Node.js 直接启动生产/开发核心服务。
- `npm run docs:dev`: 构建和运行基于 VitePress 的本地项目文档。
